NOTAS PARA A PRÓXIMA IA (CONTINUAÇÃO DO TRABALHO):
================================================================
- **PRÓXIMO PASSO SUGERIDO:** A Fase 5.2 está completa. O próximo passo é a **Fase 5.3: Playback**, que consiste em carregar os dados de um 'Ghost' do Firestore e executar as suas ações na metade do ecrã do adversário.

- **DINÂMICA DA NOVA UI (IMPORTANTE):** A UI da barra de ações foi completamente refeita e é controlada por JavaScript para um alinhamento e dimensionamento perfeitos.
  - O `index.html` contém um `<div id="button-container">` que agrupa os 9 botões.
  - A função `resize()` no `game.js` é o cérebro de tudo:
    1. Ajusta a largura do `button-container` para corresponder exatamente à base do campo de jogo.
    2. CALCULA a largura de um único botão com base na largura do contentor, padding e gap.
    3. DEFINE o `font-size` dos botões de forma proporcional à largura calculada.
  - O CSS (`style.css`) foi simplificado para apenas estruturar os elementos, deixando o dimensionamento da fonte para o JS.
  - Os botões de menu "T" e "M" foram substituídos pelos ícones "♜" e "♞".

- Manter o código modular no game.js.
- Priorizar performance para dispositivos móveis (evitar loops pesados).
- As torres e unidades devem usar o estilo de Mix Tecnológico (Medieval/Steam/Futuro).
- Sempre que completar uma tarefa, marcar com [x] (completo) ou [~] (parcial) neste ficheiro.
